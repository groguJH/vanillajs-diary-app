일기 작성, 목록 확인, 월간 달력 확인을 하나의 흐름으로 사용할 수 있는 멀티페이지 다이어리 웹 프로젝트입니다.  
Vanilla JavaScript로 `iframe` 기반 페이지 전환, `localStorage` 저장, 달력 렌더링, 테마 전환, 외부 API 연동을 직접 구현했고,
Unsplash 이미지는 서버 경유 방식으로 호출해 API 키가 브라우저에 노출되지 않도록 구성했습니다.

[🔗프로젝트 링크]:https://diary-app-tau-one.vercel.app/ 



## 프로젝트 소개

해당 프로젝트는 사용자가 일기를 작성하고, 저장된 일기를 목록과 달력 화면에서 다시 확인할 수 있도록 만든 개인 프로젝트입니다.

- 홈, 일기 작성, 목록 보기, 월간 달력 화면을 하나의 레이아웃 안에서 전환할 수 있습니다.
- 오늘의 명언을 외부 API로 불러와 메인 화면에 표시합니다.
- 홈 화면에서는 랜덤 이미지를 불러온 뒤 캔버스를 긁어보는 인터랙션을 제공합니다.
- 다크모드 토글과 커서 애니메이션으로 사용 경험을 더했습니다.
- 작성한 일기는 날짜 기준으로 `localStorage`에 저장되고, 목록과 달력 화면에서 다시 확인할 수 있습니다.
- 달력 화면에서는 날짜를 선택해 해당 일기 내용을 확인하고 일부 내용을 다시 저장할 수 있습니다.

## 기획 의도

- JavaScript만으로 여러 화면의 흐름과 상태를 관리할 수 있다.
- `localStorage`를 활용해 간단한 일기 데이터를 저장하고 다시 불러올 수 있다.
- 달력 UI와 날짜 기반 데이터 연결 방식을 직접 구현할 수 있다.
- 외부 API를 연동하되, API 키 노출 문제를 고려해 서버 경유 구조로 개선할 수 있다.

## 주요 기능

### 1. `iframe` 기반 화면 전환

- 메인 레이아웃은 유지한 채 홈, 일기 작성, 목록 보기, 월간 달력 화면을 전환할 수 있습니다.
- 상단 네비게이션 클릭만으로 다른 화면으로 이동할 수 있습니다.
- 부모 페이지의 테마 상태를 `iframe` 내부 문서에도 동기화해 전체 화면이 같은 테마를 유지하도록 구성했습니다.

### 2. 일기 작성과 저장

- 날짜, 제목, 감정, 내용을 입력해 일기를 작성할 수 있습니다.
- 입력값은 날짜를 기준으로 `localStorage`에 저장됩니다.
- 작성 완료 버튼을 누르면 목록 페이지로 이동해 저장된 결과를 바로 확인할 수 있습니다.

### 3. 목록과 모달 상세 보기

- 저장된 날짜 키를 기준으로 일기 목록을 정렬해 보여줍니다.
- 목록 항목을 클릭하면 모달로 날짜, 제목, 감정, 내용을 한 번에 확인할 수 있습니다.
- 현재 날짜 이전의 일기만 추려서 최신순으로 볼 수 있도록 구성했습니다.

### 4. 월간 달력과 날짜별 내용 확인

- 이전 달, 다음 달, 오늘 이동 버튼으로 달력을 탐색할 수 있습니다.
- 날짜 셀을 클릭하면 선택된 날짜의 저장 데이터를 모달 형태로 확인할 수 있습니다.
- 입력창에 내용을 추가로 입력하고 저장해 해당 날짜 데이터에 반영할 수 있습니다.

### 5. 홈 화면 인터랙션

- 홈 화면에서 랜덤 이미지를 불러와 캔버스에 표시합니다.
- 마우스로 캔버스를 긁는 효과를 주어 이미지가 드러나도록 구현했습니다.
- 오늘의 명언 API를 통해 간단한 동기부여 요소를 추가했습니다.

## 구현 포인트

### 1. 공통 레이아웃 유지와 `iframe` 테마 동기화

화면마다 완전히 분리된 페이지로 이동하는 대신, 메인 레이아웃 안에 `iframe`을 두고 내부 페이지만 교체하는 구조로 구성했습니다.  
이 과정에서 다크모드 상태가 부모 문서에만 머물지 않도록, `load` 시점에 `iframe` 문서의 `data-theme` 속성도 함께 바꿔 일관된 UI를 유지했습니다.

### 2. 날짜 기반 `localStorage` 구조

일기를 별도 배열로 관리하지 않고, `YYYY-MM-DD` 형식의 날짜를 키로 사용해 저장하도록 구성했습니다.  
이 방식으로 작성 페이지, 목록 페이지, 달력 페이지가 같은 저장 구조를 공유할 수 있어 데이터 흐름을 단순하게 유지할 수 있었습니다.

### 3. 달력 렌더링과 날짜 선택 흐름

현재 연도와 월 기준으로 시작 요일과 마지막 날짜를 계산해 직접 달력 테이블을 생성했습니다.  
또한 날짜 셀 클릭 시 선택 상태, 모달 표시, 저장 데이터 조회가 연결되도록 구현해 단순 조회를 넘어서 날짜별 기록 확인 경험을 만들었습니다.

### 4. API 키 보안 개선

처음에는 홈 화면 이미지 기능에서 외부 API 키가 클라이언트에 직접 노출되는 구조였기 때문에, 이를 `api/searchImg.js` 서버 함수로 분리했습니다.  
현재는 클라이언트가 `/api/searchImg`만 호출하고, 실제 `UNSPLASH_API_KEY`는 서버에서만 사용하도록 바꿔 배포 환경에서도 키 노출을 줄일 수 있도록 구성했습니다.

### 5. 사용자 경험을 위한 작은 인터랙션

명언 출력, 다크모드, 커서 애니메이션, 캔버스 긁기 효과처럼 핵심 기능 외에도 사용자가 화면을 더 흥미롭게 느낄 수 있는 요소를 추가했습니다.  
기능 구현뿐 아니라 프로젝트를 실제로 만져보는 경험까지 고려해본 점이 이 프로젝트의 차별점이라고 생각합니다.

## 실행 방법

### 1. 패키지 설치

```bash
yarn install
```

### 2. 환경변수 설정

프로젝트 루트에 `.env.local` 파일을 만들고 Unsplash 키를 등록합니다.

```bash
UNSPLASH_API_KEY=your_unsplash_access_key
```

### 3. 개발 서버 실행

```bash
yarn dev
```

### 4. 빌드

```bash
yarn build
```

## 배포 시 참고사항

- Vercel 환경변수에 `UNSPLASH_API_KEY`를 등록해야 합니다.
- 클라이언트는 `/api/searchImg`만 호출하고, 실제 키는 서버 함수에서만 사용합니다.
- 명언 API는 공개 엔드포인트를 직접 호출하는 구조입니다.
- `localStorage`를 사용하므로 브라우저 환경에서 동작합니다.

## 느낀 점

이 프로젝트를 통해 여러 화면을 iframe으로 연결하고 날짜 기준으로 데이터를 재사용하는 흐름을 직접 설계해볼 수 있었습니다.  
또한 테마 동기화, 달력 렌더링, API 키 보안 등 기능적인 부분도 고민할 수 있었고
로컬 환경에서의 동작을 넘어 라이브 환경에 사이트를 게시하여 배포하는 환경까지 함께 생각하는 습관이 중요하다는 점을 배웠습니다.
